# 任务8：图标化敌情预告与克制提示

## API

`src/game/ui/enemyIntel.js` 导出：

```js
buildEnemyIntel(preview, {
  enemies: ENEMY_CATALOG,
  towers: TOWER_CATALOG,
}, options)
```

输入只依赖波次的 `preview.icons`、敌人目录和炮塔目录，不创建Phaser对象，也不读写游戏状态。

主要输出：

- `variant`：`standard`、`boss`或`empty`。
- `badge`、`headline`、`summary`、`advice`和整体威胁颜色。
- `items`：最多4个窄屏图标项。
- `overflow`：超过上限时返回“其余N类”和隐藏单位总数。
- `accessibleLabel`：供无障碍描述或调试日志使用。

每个图标项包含：

- 敌人纹理、裁切名称、完整名称、数量和威胁颜色。
- 克制炮塔类型、纹理、裁切名称和完整名称。
- 裁切后的短提示与完整提示。
- 敌人或克制配置缺失标记。

## 窄屏规则

1. 相同敌人类型先合并数量，避免重复占格。
2. 普通预告最多显示4类，超出后显示“其余N类”。
3. Boss默认只显示Boss主图标，使用红色、`BOSS`徽标和独立variant。
4. 中文按Unicode字符裁切并补省略号；默认名称7字、提示18字、标题14字。
5. 缺失敌人配置时使用普通尸纹理；缺失克制配置时使用机枪纹理和“通用火力”，界面不会抛错。

## 建议渲染结构

- 第一行：威胁徽标、标题、整体数量。
- 第二行：最多4个敌人图标；每格显示敌人图、`×数量`和克制炮塔小图。
- 第三行：当前选中敌人的短克制提示。
- Boss模式：放大单图标并隐藏普通四格边框，保留红色警戒条。

## 集成点

1. 休整阶段取得 `getNextWavePreview()` 后调用 `buildEnemyIntel()`。
2. Phaser场景只负责把view model转换为图片和文字，不再拼接克制逻辑。
3. 点击或触摸图标时显示该项 `hint`；桌面端可额外支持悬停。
4. HUD销毁或切场景时只需销毁渲染对象，view model没有计时器和监听器。

## 风险

- 本模块只提供纹理key，不保证对应PNG已经加载；扩展敌人和炮塔接入前仍需程序回退纹理。
- `counterTower`是设计推荐，不代表实时DPS计算；数值调整后应重新检查克制文字。
- 过度压缩提示会损失语义，390px宽度下建议保持默认18字，不要继续缩短。
