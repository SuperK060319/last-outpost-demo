# 任务1：炮塔内容扩充

## 结果

现有三座炮塔扩展为“基础＋5级”，并新增三种现代军事沙盘炮塔。首级升级仍为18补给，五次单线强化合计200；完整炮塔成本为624—636，一局可以追一条专精，但三线全满需要显著放弃第二座塔和基地建设。

为避免新炮塔在行为尚未实现时进入现有菜单后完全不攻击，本轮采用分阶段目录：

- `TOWERS`：当前已接入战斗的机枪、迫击炮和狙击塔。
- `EXTRA_TOWERS`：新内容配置，暂不出现在现有建造菜单。
- `TOWER_CATALOG`：六塔完整目录，主分支完成行为分发后使用。
- `towerStats(slot)`：已经支持六塔配置。

## 新炮塔行为API

### 火焰喷射器 `cone-burn`

- 185px短射程、92px扇区。
- `power` 是每次喷射伤害，`cadence` 是伤害跳间隔，`payload` 是最大覆盖目标数。
- 命中刷新1.5秒燃烧，每0.5秒结算一次，伤害为当前威力的35%。

### 自动榴弹炮 `burst-aoe`

- 选择敌人最密集的位置，38px小爆炸范围。
- `payload` 表示每轮榴弹数，榴弹之间间隔110ms，单发飞行360ms。
- 定位介于迫击炮和机枪之间：范围更小，但连爆更稳定。

### 反装甲火箭 `rocket-armor-break`

- 优先护甲最高、其次生命最高的目标，飞行520ms。
- `payload` 表示本次伤害忽略的护甲比例。
- 命中刷新2.6秒破甲，使目标额外降低18%防护。

## 数值与经济

- 新塔价格：火焰喷射器28、自动榴弹炮30、反装甲火箭36；初始42补给无法同时购买任意两座新塔。
- 升级成本：18、26、38、52、66。
- 单线满级：建塔后再投入200。
- 完整三线：总成本624—636，符合500—650的内容目标。
- 现有三塔基础DPS小幅提高，降低第一关早期失败率；敌人和波次没有改动。

## 程序纹理

`src/game/content/extraTowerTextures.js` 导出 `createExtraTowerTextures(scene)`，会按纹理key缺失情况生成三张程序回退图。本轮没有修改BootScene或`createTextures.js`，因此不会改变当前正式素材加载。

## 主分支集成点

1. 在BootScene创建回退纹理，但正式PNG仍可使用相同key覆盖。
2. 建造菜单由`TOWERS`切换为`TOWER_CATALOG`前，先把六个选项改成分页或滚动，否则竖屏弹窗放不下。
3. 战斗分发根据`behavior.id`增加`cone-burn`、`burst-aoe`、`rocket-armor-break`三个处理器。
4. 火焰喷射器必须做射程检查；榴弹炮必须按半径伤害；火箭在敌人没有护甲字段时按0护甲结算。
5. 敌人死亡或场景重开时清理燃烧、破甲计时器，避免状态残留。

## 风险

- 现有狙击穿透仍按整数目标增长，首级在多目标场景提升明显，但面对单Boss没有收益，因此属于情境选择而非全局必选。
- 当前总资源产出较高，仅靠炮塔价格无法数学上禁止极端玩家三线全满；若试玩仍能轻松满三线，应在主经济轮调整波间奖励或补给站，而不是继续抬高首级价格。
