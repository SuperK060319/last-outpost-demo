# 任务6：休整期出售与更换炮塔

## 模块设计

出售分成两层：

1. `towerResale.js` 只计算已投入费用、返还额和当前阶段是否允许。
2. `GameScene.js` 负责弹窗确认、增加补给、清空槽位与进入同一槽位的建造菜单。

数据流为：炮塔建造费 + 三条升级线已支付的费用 → 乘以 60% → 向下取整 → 确认后加入补给。

## 安全规则

- 只有 `preparation` 和 `rest` 可拆除或更换。
- 战斗期升级面板会显示锁定说明，两个操作按钮不可点击。
- 点击“更换炮塔”不会立即删除旧塔；必须在二次弹窗确认拆除。
- 取消确认会回到原炮塔的升级面板，炮塔、升级和补给都不变。
- 确认更换后，旧塔已按明示金额返还；如果在后续建造菜单中返回战场，该槽位保持为空。
- 确认拆除时同时清空 `weaponSlots[index]`、`turrets[index]` 和 `weaponRuntime[index]`，避免旧塔射击计时残留。

## 调用与修改

升级面板中的“拆除返还”用于单纯变现，“更换炮塔”在确认后自动打开同一槽位的六塔建造菜单。

返还比例只在 `towerResale.js` 的 `TOWER_RESALE_RATE` 修改。升级费用仍由 `towers.js` 提供，不要在场景内重复写一份价格。

验证命令：

```powershell
node scripts/tower-resale-sim.mjs
npm run build
```

模拟覆盖基础塔、不同升级组合、满级三线、空槽、战斗期锁定与取消前的纯计算不变性。
