# 实时难度降速调整

## 模块设计

本轮只解决“长路线在画面上看起来却几秒跑完”的实时难度问题，不改炮塔数值、资源系统和动态难度。

数据流程为：

1. `enemies.js` 保存各波普通尸的目标旅行时间，以及跑尸、装甲尸、尸群和 Boss 的时间规则。
2. `GameScene.js` 使用实际出生点到闸门接敌线的距离，根据“距离 ÷ 目标秒数”反算每秒移动速度。
3. `waves.js` 只处理每批的出生数量与间隔，不再隐式影响单个敌人的旅行速度。
4. `scripts/wave-sim.mjs` 同时校验旅行时间、最后一批抵门余量、3秒抵达峰值和前两波基础构筑结果。

## 难度前后差异

- 改前：普通尸约 3.5 秒走完道路，第1波基础机枪近似模拟只能击杀 3/5，闸门归零。
- 改后：第1波普通尸 8.8 秒，第4波逐步缩短到 7.2 秒；跑尸 6.0–6.6 秒，装甲尸 10.2–10.9 秒，Boss 11 秒。
- 第1波：仅建造基础重机枪、不强制最优升级时，近似模拟击杀 5/5，闸门剩余 148/160。
- 第2波：保留跑尸，普通尸从 6 只降为 5 只；主力批间隔从 1.05 秒增加到 1.5 秒，收尾批从 11 秒延后到 14 秒。机枪威力 I + 基础迫击炮近似模拟击杀 6/6，闸门剩余 128/160。
- 第3、4波保留全部敌人类型和数量，压力名额分别为原比较基线的 0.90 和 0.87。
- 所有波的最后一批都会在波次时限前抵达闸门，余量为 6.75–24 秒。

## 验收标准

- 第1波普通尸从顶部到接敌线的时间约为 8.8 秒。
- 只建造一座基础重机枪时，第1波不应轻易失败。
- 第2波使用重机枪与迫击炮，并投入一次低级机枪威力升级时，应稳定清场。
- 跑尸仍明显快于普通尸，装甲尸仍明显慢于普通尸。
- 第2–4波仍保留对应的特殊敌人类型，不使用动态难度、复活或暗改玩家数值。

## 以后怎样调

安全可改区：

- 想让整波移动更慢或更快：改 `ENEMY_TRAVEL.normalByWave`。
- 想拉开特殊敌人的速度差：改 `ENEMY_TRAVEL.profiles`的倍率和上下限。
- 想减少同时抵门：改 `waves.js` 中对应批次的 `startMs` 和 `intervalMs`。

不要随便改的核心逻辑：`GameScene.js` 中从实际距离反算速度的公式。布局修改后它会自动适配新路程，避免再出现路线变长、速度仍按旧像素值运行的问题。

每次调整后运行：

```powershell
node scripts/wave-sim.mjs
npm run build
```
