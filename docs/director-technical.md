# LINE ZERO 技术兼容说明

## 本轮边界

本轮只处理浏览器外壳、音频解锁和本地战绩容错，不改变战斗数值、炮塔逻辑与关卡节奏。

## 验证矩阵

| 环境 | 目标 | 结果 |
| --- | --- | --- |
| 390×844 竖屏 | 9:16完整显示、扣除安全区、禁止画布滚动 | CSS规则已覆盖，待真机刘海复验 |
| 540×960 | 完整画布、无底部裁切 | 高度约束连续计算，不再依赖单一850px断点 |
| 手机横屏≤520px高 | 不展示缩小到不可读的战场 | 显示竖屏引导层 |
| 桌面矮窗口 | 画布按剩余高度连续缩放 | CSS规则已覆盖 |
| Chrome / Edge | 首次鼠标、触摸或按键解锁声音 | Promise安全，失败不阻断游戏 |
| iOS Safari | 用户手势内恢复AudioContext | 代码路径已覆盖，仍需真机确认 |
| 隐私/禁用存储 | 结算不因localStorage异常中断 | 写入异常被隔离 |
| 旧存档异常值 | 战绩保持非负安全整数 | 读取及结算前均归一化 |

## 仍需主流程验证

- 真机刘海、底部手势条与微信内置浏览器的实际安全区。
- 手机横竖屏切换后的Phaser画布重算是否有一帧闪烁。
- Safari首次触摸后的真实发声、静音开关和后台恢复。
- 键盘焦点、44px触控命中区与Canvas回退画面由战斗/UI模块后续处理。
