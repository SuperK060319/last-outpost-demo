# 经济与成长容错回调

## 目标

- 开局预算只够一座24补给炮塔与一次18补给初级强化。
- 第2波开始时，三种房间都能再部署一座廉价炮塔、强化一次，并预留一级闸门升级费用。
- Boss前形成“一座明确主力 + 一座中等塔”，但不能把两座塔全部升满。
- 补给站始终缓慢产出，同时保留波次运输，避免切换为持续补给后破坏原购买节点。

## 中间曲线

| 项目 | 调整后 |
| --- | --- |
| 初始补给 | 42 |
| 补给站持续产出 | 0.20 / 0.35 / 0.55 / 0.80 每秒 |
| 补给站每波产出 | 17 / 36 / 56 / 78 |
| 补给站升级成本 | 20 / 36 / 56 |
| 一级闸门 | 160 HP |
| 闸门升级成本 | 20 / 56 / 112 / 200 |
| 后勤前哨 | 每秒额外+0.05、每波额外+6 |
| 空投据点 | 初始额外+5 |

空投奖励从20降到5不是削弱总经济，而是防止该房间开局直接跳过“双塔形成”的成长节点；它仍比其他房间多5点即时操作余量。

## 模拟口径

[`economy-sim.mjs`](../scripts/economy-sim.mjs)直接读取房间、敌人、波次、炮塔建造费与 `TOWER_UPGRADE_COSTS`。升级成本扩展到5级后不需要修改脚本。

模拟会覆盖每种房间的两条路线：不投资补给站，以及按解锁时间持续投资补给站。准备、战斗和休整时间均计入持续产出；击杀收益按照当前敌人倍率和尸群数量计算，Boss奖励不计入Boss战前预算。

验收条件：

1. 所有房间开局都能购买“一塔+一次初强”，但买不起两座廉价塔。
2. 所有路线第2波均覆盖第二座廉价塔、一次初强和20补给修门预留。
3. Boss战前剩余预算覆盖主塔继续强化与副塔一次强化。
4. Boss前总收入不足以让两座塔的三条成长线全部满级。
5. 投资补给站在Boss前必须获得正净收益。

运行：`node scripts/economy-sim.mjs`，随后运行 `npm run build`。
